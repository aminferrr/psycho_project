# Тестирование продукта MindCare

## Общая информация

MindCare — мобильное приложение психологической поддержки, разработанное на платформе Flutter.  
В рамках проекта реализована система автоматического тестирования для проверки бизнес-логики, взаимодействия модулей и пользовательских сценариев.

---

## Общие принципы тестирования

- Исчерпывающее тестирование невозможно, поэтому проверяются наиболее критичные сценарии.
- Тестирование зависит от контекста приложения и его функциональных требований.
- Основной акцент сделан на стабильность работы, корректность логики и интерфейса.

---

## Уровни тестирования

1. Модульное тестирование (Unit Testing)  
   Проверка отдельных функций и методов приложения без взаимодействия с внешними сервисами.

2. Интеграционное тестирование (Integration Testing)  
   Проверка взаимодействия репозиториев с сервисами и корректности передачи данных.

3. Системное тестирование (UI / End-to-End Testing)  
   Проверка пользовательского интерфейса и ключевых сценариев работы приложения.

---

## Как запускать тесты

### Unit и Integration тесты
Выполнить команду:
flutter test

### UI / End-to-End тесты
Выполнить команду:
flutter test integration_test

---

## Структура тестов

Unit тесты располагаются в директории:
test/unit/

Файлы:
- constants_test.dart
- anon_generator_test.dart

Integration тесты располагаются в директории:
test/integration/

Файл:
- forum_repository_test.dart

UI / End-to-End тесты располагаются в директории:
integration_test/

Файл:
- login_screen_e2e_test.dart

---

## Покрытые пользовательские сценарии

1. Запуск приложения и отображение экрана входа.
2. Проверка обработки некорректного email при авторизации.
3. Переключение между режимами входа и регистрации.

---

## Реализованные тесты

### Unit тесты (4)

- Проверка названия приложения (AppConstants.appName).
- Проверка корректности констант коллекций Firestore.
- Проверка генерации случайного профиля пользователя.
- Проверка генерации случайного аватара пользователя.

---

### Integration тесты (3)

- Проверка добавления поста форума.
- Проверка получения списка постов форума.
- Проверка добавления комментария к посту.

---

### UI тесты (3)

- Проверка корректной загрузки экрана входа.
- Проверка отображения ошибки при неверном email.
- Проверка переключения режима входа и регистрации.

---

## Используемые инструменты тестирования

- flutter_test
- integration_test
- mocktail
- fake_cloud_firestore

---

## Использование моков

- FirestoreService замокирован с использованием mocktail.
- AuthService замокирован в UI тестах.
- Fake Firebase Firestore используется для эмуляции базы данных.

Использование эмуляции базы данных позволяет выполнять тестирование без подключения к реальному серверу Firebase.
